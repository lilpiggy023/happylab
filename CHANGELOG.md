# 素材打标台产品更新日志

## v1.0 - 产品新增功能

v1.0 完成了素材打标台从 0 到 1 的基础闭环，核心目标是让投流素材可以从上传、拉片、自动打标到人工确认入库，形成可持续沉淀的本地素材资产库。

### 1. 素材上传与拉片基础流程

- 支持上传视频素材，并在本地生成素材记录。
- 接入 Video AI Analysis Skill V1 的素材解析能力，生成关键帧、故事板、音频、基础 metadata 等拉片素材包。
- 上传并拉片后的素材会保留记录，前端展示素材缩略图、素材名称、处理状态与相关产物链接。
- 支持素材卡片删除，便于清理无效或测试素材。

### 2. AI 拉片报告生成

- 接入 PocketCity 模型接口，使用 `gemini-3.1-pro-preview` 作为 AI 拉片与 AI 打标模型。
- 前端支持输入模型 Key。
- 从“假报告/素材包说明”升级为正式 AI 双语拉片报告生成。
- AI 拉片基于抽帧、故事板、音频转写字幕等信息进行内容分析。
- 报告支持 Markdown 格式，前端可上传 `.md` 拉片报告用于后续打标。

### 3. 音频转写与字幕展示

- 将音频转写与报告生成拆分为独立步骤。
- 支持大模型转写与本地 ASR 转写两种方式。
- 本地 ASR 支持选择转写语种，至少覆盖中文、英语、印尼语。
- 支持本地 faster-whisper 模型兜底，避免完全依赖云端转写。
- 对非中文字幕补充中文版翻译。
- 字幕区支持“原文/中文”切换，默认展示中文。

### 4. 自动打标与人工确认

- 自动打标页支持选择已有素材，自动带入该素材的拉片报告路径。
- 支持上传 Markdown 拉片报告并触发 AI 自动打标。
- 自动打标结果支持人工编辑、增删标签与备注。
- 人工确认后，素材标签可正式入库。
- 已入库素材在标签库中沉淀展示。

### 5. 标签体系与标签库管理

- 建立一级标签、二级标签的基础标签体系。
- 支持手动新增一级标签、二级标签。
- 支持为标签维护定义，自动打标时会把标签定义提供给 AI 参考。
- 鼠标悬停标签可查看定义。
- 点击标签可修改名称、定义或删除。
- 支持一级标签删除。
- 增加标签墓地，用于保存已删除标签与删除原因。
- 删除标签时要求二次确认并填写删除原因。
- AI 自动打标时同时参考标签库和标签墓地，避免复用已删除或近义重复标签。

### 6. Prompt 配置

- 新增 Prompt 配置页，可查看和修改 AI 拉片、AI 打标使用的 skill/prompt。
- 支持恢复默认 Prompt。
- Prompt 与标签库联动，用于控制 AI 打标时对已有标签、新增标签、标签定义的输出规则。

### 7. 前端体验基础优化

- 上传拉片页以素材卡片形式展示素材。
- 素材卡片展示 storyboard 缩略图、素材名称与状态。
- 自动打标页左侧展示当前素材、抽帧、字幕、文件产物等信息。
- 抽帧缩略图限制固定尺寸，避免溢出页面。
- 错误提示改为居中 toast 展示。

## v1.1 - 产品更新

v1.1 重点升级了标签体系的层级能力、素材库检索能力、全流程自动化体验，以及局域网部署能力，使产品更接近可多人协作使用的素材资产管理工具。

### 1. 标签体系升级为多层路径结构

- 将原本“一级标签 / 二级标签”的结构升级为“一级标签 / 多层子标签路径”。
- 每个一级标签可独立设置子标签层级数，最多支持 5 层子标签。
- 子层级标签不与上一层强绑定，可作为同一一级标签下某一层级的自由标签池。
- 自动打标输出格式升级为标签路径，例如：`强冲突 - 言语辱骂 - 反派威胁女主与幼童`。
- 自动打标页中，标签路径以链式胶囊展示，子标签之间用连线连接。

### 2. 标签库多层管理能力

- 标签库样式升级为按一级标签卡片展示，并在卡片内分层展示子标签。
- 支持修改子层级名称，例如将“二级”改为“身份/角色”。
- 子层级标题颜色增强，提升层级辨识度。
- 一级标签支持点击打开卡片，修改一级标签名称、定义或删除。
- 一级标签区域与子标签样式做了区分，减少视觉混淆。
- 一级标签支持长按空白区域拖动排序，修改名称或定义不再自动移动到最后。

### 3. 子标签合并与转移

- 支持同层级子标签合并，合并后以目标标签的名称和定义为准。
- 修复同层级合并误报“目标标签不在当前层级”的问题。
- 子标签支持自由转移到任意一级标签下的任意子层级。
- 保留长按拖动转移交互：长按子标签后，标签跟随鼠标移动，拖到目标层级区域后松开并确认转移。
- 拖动时目标层级区域高亮，提升转移操作的可感知性。

### 4. AI 自动打标增强

- 自动打标时同时参考多层标签库与标签墓地。
- AI 新生成的所有标签都要求带定义，包括各层级子标签。
- 修复“人设组合”与已重命名一级标签重复新建的问题，避免近义一级标签重复产生。
- 修复全流程任务中自动打标未自动完成的问题。
- 标签预览区鼠标悬停标签时展示标签定义。

### 5. 自动打标页交互优化

- 删除重复的自动打标按钮，将主操作集中到右侧标签预览区域。
- 抽帧区展示所有抽帧，而不是固定 8 帧。
- 点击抽帧不再跳转新页面，改为当前页面弹出放大预览卡片。
- 左侧信息排序调整为：抽帧、字幕、文件。
- 产物与音频合并到文件区展示。
- 报告区整合“生成报告 / 重新生成报告 / 上传拉片报告”操作。
- 字幕区新增“重新转写”按钮，重新走“选择转写方式 - 选择语言 - 大模型翻译”的完整流程。
- “原文/中文”切换改为图标加当前语种的按钮样式。

### 6. 素材库升级

- 【上传拉片】页面更名为【素材库】。
- 【上传拉片】按钮更名为【上传素材并分析】。
- 素材库顶部新增拉片任务区，展示排队和处理中任务的进度、状态与错误信息。
- 点击素材卡片可跳转到对应素材的自动打标页。
- 素材卡片精简为缩略图、素材名称与状态。
- 素材库增加搜索能力，并拆分为：
  - 素材名搜索：通过输入关键词筛选素材名。
  - 标签搜索：支持输入关键词，也支持打开下拉栏按一级标签和层级选择标签。
- 标签下拉支持输入关键词缩小标签范围。
- 支持清空搜索条件。

### 7. 上传与全流程自动化

- 点击【上传素材并分析】后弹出上传视频弹窗。
- 删除素材名称手动填写栏，素材名称由上传文件和任务信息自动生成。
- 点击开始后自动执行完整流程：解析抽帧拆音频、转写并翻译字幕、AI 拉片、自动打标。
- 支持拖动视频到上传框完成上传，不再触发浏览器自动打开新标签页。
- 拖入视频时有高亮反馈，松手后自动填入文件。

### 8. 局域网部署

- 服务启动地址从仅本机 `127.0.0.1` 扩展为监听 `0.0.0.0`。
- 支持同一局域网设备通过本机 IPv4 地址访问素材打标台。
- 启动时自动打印本机访问地址与局域网访问地址。
- 保留 `PORT`、`HOST` 环境变量配置能力，方便后续部署调整。

### 9. 备份与交付

- 生成 v1.0 桌面备份包：`素材打标台-260624-v1.0`。
- 生成 v1.1 桌面备份包：`素材打标台-260624-v1.1.zip`。
- v1.1 备份包包含：项目本体、素材数据、本地 faster-whisper 模型、本地 Vosk 模型与辅助脚本。

## v1.2 - 产品更新

v1.2 重点修复了多层标签体系在真实使用中的一致性问题，并进一步优化素材库筛选、模型 Key 前置提醒、标签墓地审计与标签编辑稳定性，让产品从“能跑通”升级到更适合长期维护和团队协作的版本。

### 1. 模型 Key 前置校验与提示体验

- 建立统一的模型 Key 前置校验机制，所有需要调用大模型的入口都会在点击开始时先校验 Key。
- 无模型 Key 时，【上传素材并分析】、【开始拉片】、【自动打标】、【生成报告】等按钮进入灰态提示，不再等流程跑到一半才报错。
- 点击灰态按钮会弹出 toast：`请先输入模型Key`，并高亮左侧模型 Key 输入区域。
- 优化模型 Key 高亮 UI，改为整块区域柔和描边、焦点环与“需要模型 Key”提示标签。
- `请先输入模型Key` 提示固定显示在对应按钮正下方，并修复提示导致按钮与标题不对齐的问题。
- 保留本地 ASR 的无 Key 使用能力，仅在选择“大模型”转写方式时要求模型 Key。

### 2. 素材库标签筛选体验升级

- 素材库标签筛选从列表式下拉升级为胶囊标签池，视觉风格与【标签库】保持一致。
- 标签筛选按一级标签分组展示，并在一级标签下按子层级分区展示标签胶囊。
- 一级标签支持折叠与展开，降低标签数量较多时的浏览压力。
- 输入关键词搜索时会自动展开匹配的一级标签，便于快速选择。
- 已选中标签胶囊会高亮，提升当前筛选条件的可见性。
- 修复点击一级标签折叠时下拉栏自动关闭的问题。
- 优化标签下拉栏定位，使面板向左展开并保留右侧安全距离，避免贴到浏览器边缘。

### 3. 标签墓地与合并审计增强

- 标签墓地支持清空前备份，清空操作前会将墓地信息单独导出为备份文件。
- 已清空历史标签墓地数据，并保留独立备份文件。
- 子标签合并时，被合并的源标签会写入标签墓地。
- 合并操作新增合并原因输入，墓地记录会保留源标签、层级、定义、合并目标和合并原因。
- 标签墓地展示中补充“已合并到”信息，便于后续追踪标签治理历史。

### 4. 一级标签重命名与层级字段一致性修复

- 修复一级标签改名后出现两个相近一级标签的问题。
- 一级标签重命名现在会同步迁移 `customSchema`、`levelTags`、`categoryLevels`、`categoryLevelNames`、`categoryOrder`、定义、素材引用和墓地引用。
- 当重命名目标一级标签已存在时，后端会执行合并，而不是拆分出两个一级标签。
- 将历史长名 `人设组合（身份/角色配置 + 气质/状态 + 人物关系 + 关系张力）` 合并回短名 `人设组合`。
- 清理历史残留的 `人设组合（身份/角色配置 + 气质/状态 + 核心关系 + 关系张力）` 元数据。
- 修复 `人设组合` 在后端仍被规范化回旧长名的问题，将短名设为正式规范名，旧长名仅作为别名兼容。

### 5. 子标签编辑稳定性修复

- 修复三级标签改名后误在二级标签下新增同名标签的问题。
- 修复删除三级标签时误报“该子标签不在当前层级，无法删除”的问题。
- 修复修改三级标签时误报“原二级标签不存在 / 原四级标签不存在”的问题。
- 子标签改名时，如果前端传入的层级已过期，后端会自动扫描真实层级并纠偏；若标签只存在于一个层级，则直接按真实层级保存。
- 修复同层级标签合并时误报“目标标签不在当前层级，不能合并”的问题。
- 子标签新增后无需刷新即可在标签库中展示。

### 6. 标签库交互与弹窗体验优化

- 标签编辑卡片采用更稳的关闭逻辑：从卡片内拖选文字到外部松开时，弹窗不会误关闭。
- 点击遮罩仍可关闭弹窗，保持轻量交互。
- 标签库中一级标签和子标签编辑继续支持改名、改定义、删除、合并和转移。
- 修复一级标签改名与定义保存后排序异常或数据分裂的问题。

### 7. 备份与交付

- 生成 v1.2 桌面备份包：`素材打标台-260626-v1.2.zip`。
- v1.2 备份包包含：项目本体、素材数据、本地 faster-whisper 模型、本地 Vosk 模型与相关模型压缩包。
- 关键数据修复前已生成标签库备份，便于回滚排查。

- v1.1：完成多层标签体系、自动化拉片打标流程、素材库检索与局域网部署能力升级。
- v1.2：完成标签治理审计、模型 Key 前置校验、素材库标签筛选体验和多层标签一致性修复。
